# tuning_records

This folder stores one markdown note per tuning round.

Expected contents for each note:

- target run / checkpoint
- previous eval folder path
- pre-tuning eval summary
- WandB observations
- exact config changes
- rationale
- expected effect
- suggested next run name / run suffix
- follow-up result if available

Recommended naming pattern:

- `YYYY-MM-DD_short_description.md`
